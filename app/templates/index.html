      🔌 API Endpoints

      MethodEndpointDescription
      GET/Browser UI (this page)
      GET/healthHealth check — returns {"status":"healthy"}
      GET/programsAll 3 ACEest programs with workout & diet plans
      GET/programs/FLSingle program by code (FL / MG / BG)
      POST/caloriesCalculate daily kcal: {"weight_kg":75,"program_code":"FL"}
      GET/workoutsList all logged workouts
      POST/workoutsLog a workout: {"name":"...","duration_minutes":60}
      GET/membersList all registered members
      POST/membersRegister a member: {"name":"...","email":"..."}
      GET/gym-infoGym capacity, area, break-even metrics

      📜 Version History (10 Desktop → 1 Web API)

      The application evolved from Tkinter desktop scripts to this Flask web service deployed via Docker and CI/CD.

      • Aceestver-1.0 — Tkinter GUI, 3 programs (FL/MG/BG), workout & diet display, gym metrics
      • Aceestver-1.1 — Added calorie factor formula, colour-coded programs, CSV export
      • Aceestver-2.1.2 — SQLite database for client profiles and weekly progress tracking
      • Aceestver-2.2.1 — Matplotlib charts for adherence visualisation
      • Aceestver-2.2.4 — Height/weight schema, membership expiry, full DB migration logic
      • Aceestver-3.0.1 — Tabbed UI (Client Summary + Progress & Analytics), AI program generator, PDF reports
      • Aceestver-3.1.2 — Role-based login screen (Admin / Trainer), full workout & exercise logging
      • Aceestver-3.2.4 — Multi-table DB (users, clients, workouts, exercises, metrics), membership billing
      • Aceestver1.1.2 — Multi-client table UI, progress chart, notes field, CSV bulk export
      • Aceestver2.0.1 — Clean DB-backed final desktop baseline before web migration
      • app.py (this) — Flask REST API + Bootstrap UI + Docker + GitHub Actions + Jenkins ✅
